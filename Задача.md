## Общее описание
Разработайте SPA приложение для системы управления заявками с использованием **TypeScript 5** и одного из фреймворков: **React 18** (с использованием Vite) или **Vue 3** (Composition API + Pinia). Приложение должно включать три основных экрана: аутентификация, список заявок и детальная информация о заявке.

## Технические требования
**Обязательно:**  Стили использовать максимально приближенные к дизайну заказчика на основе предоставленных заказчиком макетов в Figma.

### Стек технологий
- **Обязательно:** TypeScript 5, React 18 (Vite) или Vue 3 (Composition API)
- **Рекомендуется:** 
  - Для React: Zustand/Jotai + React Router 6.4
  - Для Vue: Pinia + Vue Router 4
- **Веб-сервер:** любой на nodejs или фреймворки Next.js / Nuxt.js
- **HTTP-клиент:** Axios или нативный Fetch (с типизацией ответов)
- **Стилизация:** CSS Modules/Tailwind CSS или аналоги с поддержкой темизации
- **Сборка:** Vite с продакшен-оптимизациями

### Архитектура приложения
```
src/
├── @types/             # TypeScript-типы
├── api/               # Сервисы и API-клиент
├── assets/            # Статические ресурсы
├── components/        # Переиспользуемые компоненты
├── constants/         # Константы и enum'ы
├── hooks/             # (React) / composables/ (Vue) - логика
├── layouts/           # Макеты страниц
├── lib/               # Вспомогательные утилиты
├── pages/             # Страницы приложения
├── store/             # Состояние приложения
├── styles/            # Глобальные стили
├── utils/             # Функции форматирования и обработки данных
└── App.tsx            # Корневой компонент
```

### Требования к коду
1. Полная типизация всех props, API-ответов и состояний
2. Разделение ответственности между компонентами и бизнес-логикойъ
3. Обработка ошибок и edge cases
4. Использование lazy loading для роутов
5. Реализация аутентификации через Bearer Token
6. Использование guards в маршрутизации (например, для проверки доступа и авторизации)


## Функциональные требования

### 1. Аутентификация
- **Эндпоинт:** `POST https://gocrm.gruzoperevozki-rf.com/crm/auth`
- **Тестовые данные:** 
  ```json
  { "login": "test", "password": "12345678" }
  ```
- **Требования:**
  - Валидация полей на клиенте
  - Хранение JWT-токена в `localStorage` с возможностью очистки при logout
  - Автоматическое перенаправление на список заявок при успешной аутентификации
  - Защита роутов от неавторизованного доступа

### 2. Основной экран с навигацией
- **Требования к меню:**
  - Левая сайдбар с иконками
  - Раскрытие полного меню при наведении (`:hover`)
  - Анимация плавного появления/скрытия
  - Адаптивность для мобильных устройств

### 3. Список заявок
- **Эндпоинт:** `GET https://gocrm.gruzoperevozki-rf.com/crm/orders`
- **Требования:**
  - Табличное представление с колонками: ID, Адреса, Статус, Дата создания
  - Загрузка данных при монтировании компонента
  - Индикация состояния загрузки
  - Обработка ошибок сети и серверных ошибок
  - Пагинация (на основе данных из `pagination` в ответе)

### 4. Детальная информация о заявке
- **Требования:**
  - Отображение в модальном окне с затемнением фона
  - Форматирование JSON-данных в человекочитаемый вид
  - Кнопка закрытия по ESC и клику вне модалки
  - Адаптивный дизайн для мобильных устройств
